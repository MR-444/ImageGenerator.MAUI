    HOW A COMFYUI WORKFLOW IS USED VIA API (the user's question):
      A workflow is the node graph built on ComfyUI's canvas. For API use it is
      exported ONCE as "API format" JSON (Workflow menu > Export (API); on old
      UIs enable dev mode first). That JSON maps node-id -> {class_type,
      inputs}. The app never builds graphs — it PATCHES a few input fields in
      the template (prompt text, seed, width/height, ckpt_name) and POSTs it.
      Normal saves / PNG-embedded workflows are the UI format and are NOT
      accepted by POST /prompt.

    API surface (stable knowledge; verify against the live instance early):
      - POST /prompt {"prompt": <api-format graph>, "client_id": id}
        -> {prompt_id}; 400 carries node-level validation errors.
      - GET /history/{prompt_id} -> outputs incl. image filenames when done.
      - GET /view?filename=&subfolder=&type=output -> image bytes.
      - GET /object_info -> every installed node + its valid enum values;
        CheckpointLoaderSimple.input.required.ckpt_name[0] IS the list of
        local checkpoints (same trick for LoRAs, samplers, VAEs).
      - WS /ws?clientId= pushes progress; polling /history also works and
        matches the existing Replicate create->poll shape.
      - GET /system_stats -> cheap "is fireEngine up" health check.
