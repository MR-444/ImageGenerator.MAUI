RECENTLY DONE
-------------

[x] Faster, byte-identical PNG saves — commit 8f32b87 (2026-06-13), user-verified
    (posted images render on CivitAI after a post). New PngTextChunkWriter
    (Infrastructure/Imaging) splices a single uncompressed iTXt "Comment" chunk
    straight into the original PNG bytes: validates the signature, walks chunks,
    strips any pre-existing text/eXIf chunks (dedup — no double prompt), inserts the
    new chunk before IEND, hand-rolled CRC32 (no System.IO.Hashing dep). iTXt+UTF-8
    so emoji/non-Latin prompts survive (ImageSharp already emitted iTXt because the
    old code set language tag "en", so the reader is unchanged). ImageFileService.
    SaveImageWithMetadataAsync now fast-paths PNG output when the source is genuine
    PNG (IsPng): dimensions header-only via Image.IdentifyAsync (the GalleryService
    pattern), then splice + File.WriteAllBytesAsync — NO decode, NO re-encode, pixels
    byte-identical to the provider. Metadata-text build is byte-for-byte unchanged so
    every key + InvariantCulture value round-trips identically; readers (Remix,
    CivitaiMetaBuilder) untouched. FALLBACK to the old decode+re-encode path for
    JPG/WebP and for PNG-requested-but-source-is-not-PNG (a provider returning
    JPEG/WebP still transcodes correctly). Win: CivitAI uploads (read off disk) now
    carry the exact original image, and the heaviest op (Rgba32 decode) is skipped on
    the common PNG save. 914 tests (+10).
