ROADMAP / OUTLOOK — recorded forward plan (each = its own future task)
---------------------------------------------------------------------

User-selected 2026-06-13 from a grounded brainstorm. Respect scope rules
(no multi-image-per-click; no kontext/negative_prompt/A1111-in-saved-files).
These go INTO the README Outlook AND are the backlog for future contexts.

[ ] Remix from an image — reload a saved image's embedded recipe (prompt, seed,
    model, params) back into the generator. Reader exists: GalleryService.
    ReadMetadataAsync parses the key:value lines; this maps them back onto
    ImageGenerationParameters + selects the model. Closes the reproducibility
    loop the app is built around. FLAGSHIP.
[ ] One-click upscale of a result — route a finished image through an existing
    provider (a Replicate upscaler model, or a ComfyUI upscale workflow). New
    operation type but reuses dispatch/provider plumbing.
[ ] Saved parameter presets — name + recall a model+params combo.
[ ] Clear finished jobs — queue-eviction control (cards never evict today;
    GeneratorViewModel.Jobs). Was a Known-issues "v1.x candidate".
[ ] Configurable output folder — currently hard-pinned in
    Shared/Constants/OutputPaths.cs (GeneratedImagesDirectory =
    %USERPROFILE%\Pictures\ImageGenerator.MAUI\). Make it a setting.
[ ] Faster, byte-identical PNG saves (efficiency) — ImageFileService.
    SaveImageWithMetadataAsync fully decodes to Rgba32 just to read dimensions
    + attach a Comment chunk, then re-encodes. Splice the tEXt chunk into the
    original bytes instead (dimensions header-only via Image.Identify, as
    GalleryService already does) → faster AND keeps the API's exact bytes, so
    CivitAI uploads become truly byte-identical.

NOT selected (record so they aren't re-pitched as new): gallery delete/search/
favorites, compare/A-B view. Excluded by scope: seed-variation "more like this"
(only OK as a sequential batch, never multi-output).
