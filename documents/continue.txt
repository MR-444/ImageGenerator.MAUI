    WHY: a JSON *validator* is worthless (LLMs emit valid JSON). But Ideogram 4's
    json_prompt carries spatial BBOXES, and placing boxes on a canvas is the one
    thing an LLM is bad at. The editor adds that spatial value and replaces the
    current hand-paste-and-edit-[y,x,y,x] workflow.

    CRITICAL CONSTRAINT (confirmed in code + a prior live 422): the app goes
    through Replicate, whose ideogram-v4 cog types json_prompt as a STRING, not an
    object ("input.json_prompt: Expected: string, given: object"). So the editor
    builds a typed model and serializes to a COMPACT STRING into Parameters.Prompt
    with UseJsonPrompt=true — the existing IdeogramV4Descriptor.Build path is
    UNCHANGED (Core/Domain/Descriptors/IdeogramV4Descriptors.cs:55-71).

    TWO OUTPUTS from one model (Serialize(model, indented)):
      - Prompt box        -> compact string (this app's Replicate generation).
      - "Save JSON to file" -> pretty .json on disk
        (Pictures\ImageGenerator.MAUI\json-prompts\) — portable to a local Ideogram
        install / official API / any Ideogram-backed UI. (User-requested addition.)

    "text"; optional bbox [y_min,x_min,y_max,x_max] ints 0-1000, origin top-left,
    canvas always 1000x1000; per-element palette <=5.
